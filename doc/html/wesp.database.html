wesp.database module¶

class wesp.database.Database[source]¶

Contains all functions related to the database. To ensure that only one DB connection is used, everything in this class ist static.

The different raw statements are completed in the init_database() using the data inside of wesp.definitions.AllParameter.

The function create_database_and_table_if_not_existing() will check if the database and the table exist, otherwise it will create it using the databaseCreateStatement.

Data can be inserted using the function insert_data_set(). This function expects the data to be in the format of CLIENT_DATA inside the cli_parser module.

static create_database_and_table_if_not_existing()[source]¶

Will check the existence of the database and the table with the names specified in the init_database() function. If they do not exist they will be created. Therefore a connection to the DB Information_Schema is performed.

Parameters:config – config for database connection, can be generated by wesp.helper.generate_db_conf_from_context()
databaseCreateStatement = 'CREATE DATABASE IF NOT EXISTS %%DATABASE%%;'¶

This statement will be used to create the database. It will only trigger if the database does not exists. Note the name of the database will be inserted in the init_database() function

static init_database(config, parameter_create, parameter_insert)[source]¶

Will initialize the database and complete the SQL Statements with the given names and sub-statements

Parameters:
  • ctx – current Context
  • config – config for database connection, can be generated by wesp.helper.generate_db_conf_from_context()
  • parameter_create – parameter part of create statement, can be generated by wesp.helper.generate_parameter_create_statement()
  • parameter_insert – parameter part of insert statement, can be generated by wesp.helper.generate_parameter_insert_statement()
insertStatement = 'INSERT INTO %%TABLE%% %%PARAMETER%%;'¶

This statement is used to insert data, the name and list of parameters as well its value names are inserted in the function init_database(). The generation of that data is done in wesp.helper.generate_parameter_insert_statement() function but has to be ran by the caller of the init_database() function

static insert_data_set(ctx, time)[source]¶

Will inserted the given data into the database based on the config and statement form the init_database() function

Parameters:
  • data_set – data_set to be inserted, must contain the same fields name as defined in the insertStatement
  • ctx – current Context
static is_ready()[source]¶

Will check, if this class is ready, which means that the config has been set and the raw statements have been initialized.

Returns:True if database has been initialized, otherwise False
tableCreateStatement = 'CREATE TABLE IF NOT EXISTS %%DATABASE%%.%%TABLE%% (\n `id` int(11) unsigned NOT NULL AUTO_INCREMENT,\n `Timestamp` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,\n %%PARAMETER%%\n PRIMARY KEY (`id`),\n KEY `ipIndex` (`%%IP_INDEX%%`),\n KEY `macIndex` (`%%MAC_INDEX%%`)\n ) ENGINE=InnoDB DEFAULT CHARSET=utf8;'¶

This statement creates the table. It will only trigger if the database does not exists. Note the name of the database and table, as well as all other fields (based on registered Parameter in wesp.definitions.AllParameter) will be inserted in the init_database() function. The generation of that data is done in wesp.helper.generate_parameter_create_statement() function but has to be ran by the caller of the init_database() function
